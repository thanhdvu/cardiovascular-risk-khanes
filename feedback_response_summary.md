# 교수님 피드백 검토 자료

## 1. 교수님 피드백 검토 배경

본 branch는 과제2 제출본에 대해 교수님 피드백을 받은 후, 추가 검토 내용을 정리하기 위해 생성하였다. 기준 commit은 과제2 제출 시점의 commit인 `a221bf9021e80737e5c79348667600132923378d`이다.

원래 과제2 notebook과 report 파일은 수정하지 않고 그대로 두었다. 추가로 수행한 분석 결과는 `feedback_review/` 폴더에 따로 정리하였다. 이 문서는 모델을 방어하거나 최종 결론을 내리기 위한 자료가 아니라, 피드백에 따라 어떤 부분을 추가로 확인했는지 정리하고 추후 논의를 준비하기 위한 자료이다.

교수님 피드백에서 중점적으로 확인한 내용은 다음과 같다.

- 고혈압과 관련된 변수를 추가로 검토할 것
- class weight 보정을 적용해 볼 것
- 모델이 고혈압군을 충분히 학습하지 못하는 경우 PSM 적용을 고려할 것

## 2. 과제2 모델 재검토

과제2에서는 KNHANES 2017-2023 자료 중 19-39세 대상자를 중심으로 고혈압 위험 예측을 수행하였다. 고혈압 target은 수축기 혈압 `HE_sbp >= 140` 또는 이완기 혈압 `HE_dbp >= 90`인 경우 `hypertension = 1`로 정의하였다.

과제2 제출본 기준으로 19-39세 대상자는 10,974명이었으며, 고혈압군은 575명으로 정리되어 있었다. 모델 비교에서는 AUC를 주요 기준으로 사용하였고, Logistic Regression이 가장 높은 AUC를 보였다.

과제2 제출본의 주요 결과는 다음과 같다.

| 항목 | 결과 |
|---|---:|
| Logistic Regression AUC | 0.8099 |
| Tuned Logistic Regression AUC | 0.8118 |
| 비교 기준 | AUC |
| 최상위 모델 | Logistic Regression |

## 3. Confusion Matrix 검토

교수님 피드백을 반영하여, 모델이 대부분을 다수 class로만 예측하는지 확인하기 위해 confusion matrix를 다시 검토하였다. 특히 고혈압군 탐지 여부를 확인하기 위하여 실제 class 분포와 예측 class 분포를 함께 살펴보았다.

confusion matrix 결과는 아래 파일에 정리하였다.

- `feedback_review/figures/confusion_matrix.png`

이 검토는 모델이 고혈압군을 어느 정도 예측하고 있는지 확인하기 위한 단계이다. 다만 confusion matrix 하나만으로 모델의 적절성을 단정하기는 어렵기 때문에, precision, recall, class imbalance 처리 방식과 함께 추가 검토가 필요하다.

## 5. 고혈압 관련 변수 검토

교수님 피드백에 따라 기존 변수 외에 고혈압과 관련될 수 있는 변수를 추가로 검토하였다. 이 과정은 체계적 문헌고찰을 수행한 것은 아니며, 수업 프로젝트 범위에서 임상적으로 관련성이 있을 수 있는 변수를 확인한 것이다.

검토한 변수는 다음과 같다.

| 변수 | 검토 이유 |
|---|---|
| `HE_wc` | 허리둘레는 복부비만과 관련이 있으며, 비만은 고혈압 위험과 관련될 수 있어 검토하였다. |
| `HE_HbA1c` | HbA1c는 혈당 조절 상태를 보여주는 지표이다. 당대사 이상은 심혈관 위험과 함께 고려될 수 있어 확인하였다. |
| `HE_chol` | 총콜레스테롤은 지질 상태를 보여주는 기본 지표이며, 심혈관계 위험 요인과 관련될 수 있어 검토하였다. |
| `HE_HDL_st2` | HDL cholesterol은 지질 대사와 관련된 변수이다. 혈압 및 심혈관 위험과 함께 논의될 수 있어 포함 가능성을 확인하였다. |
| `HE_HPfh1` | 고혈압 가족력은 개인의 고혈압 위험과 관련될 수 있으므로 추가 변수로 검토하였다. |
| `mh_stress` | 스트레스는 생활습관 및 건강 상태와 관련될 수 있으며, 고혈압 위험과도 관련 가능성이 있어 확인하였다. |

후보 변수의 사용 가능 여부는 `feedback_review/tables/candidate_variable_availability.csv`에 정리하였다.

## 6. 추가 변수 적용 결과

위 변수들을 추가한 후, 기존 Logistic Regression 흐름을 유지한 상태에서 간단한 비교를 수행하였다. 결측값 처리 후 비교에 사용된 표본 수는 9,889명이었다.

| 지표 | 기존 모델 | 추가 변수 모델 |
|---|---:|---:|
| 결측 제외 후 표본 수 | 9,889 | 9,889 |
| Precision | 0.1372 | 0.1434 |
| Recall | 0.7248 | 0.7064 |
| F1 | 0.2307 | 0.2384 |
| ROC AUC | 0.7952 | 0.8084 |
| Average Precision | 0.1784 | 0.1819 |

일부 지표에서 성능 향상이 확인되었으나 개선 폭은 크지 않았다. 또한 recall은 약간 낮아지는 결과가 나타났다. 따라서 추가 변수 사용 여부에 대해서는 confusion matrix, recall, precision의 균형을 함께 보면서 추가 논의가 필요하다.

상세 결과는 `feedback_review/tables/feature_expansion_results.csv`에 저장하였다.

## 7. 클래스 불균형 처리 검토

고혈압군 비율이 낮기 때문에, class imbalance 처리 방식도 추가로 비교하였다. 비교 대상은 다음 두 가지이다.

- SMOTE + class weight 적용
- class weight만 적용

| 모델 | Precision | Recall | F1 | ROC AUC | Average Precision |
|---|---:|---:|---:|---:|---:|
| SMOTE + class weight | 0.1414 | 0.7217 | 0.2365 | 0.8194 | 0.2691 |
| class weight only | 0.1424 | 0.7217 | 0.2378 | 0.8199 | 0.2595 |

두 방식의 성능 차이는 크지 않은 것으로 확인하였다. class weight만 적용한 경우 precision, F1, ROC AUC가 아주 조금 높았고, SMOTE + class weight 방식은 Average Precision이 조금 높았다. 이 결과 역시 어느 방식이 확실히 우수하다고 단정하기보다는, 고혈압군 탐지와 false positive 문제를 함께 보며 논의할 필요가 있다.

상세 결과는 `feedback_review/tables/class_weight_comparison.csv`에 저장하였다.

## 8. PSM 적용 검토

교수님 피드백에 따라 PSM 적용 가능성도 검토하였다. 다만 PSM을 바로 적용하기 전에, 먼저 기존 모델의 confusion matrix를 확인하고, 고혈압 관련 변수 추가 및 class imbalance 처리 방식을 비교하였다.

현재 단계에서는 PSM 적용 여부를 최종적으로 판단하기보다는, 추가 변수 적용 결과와 class weight 비교 결과를 바탕으로 논의가 필요하다고 보았다. 특히 고혈압군 예측이 어느 정도 이루어지는지, precision과 recall의 균형을 어떻게 볼 것인지, PSM을 적용할 경우 연구 질문이 어떻게 달라지는지에 대한 추가 검토가 필요하다.

따라서 본 문서에서는 PSM을 결론적으로 배제하거나 채택하지 않고, 교수님 피드백에 따른 검토 항목으로 정리하였다.
